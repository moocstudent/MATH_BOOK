## Core Notes

### Basic Concepts

An (undirected) **graph** is a pair $G=(V,E)$, where $V$ is a nonempty set of **vertices** and $E$ a set of **edges**, each edge joining a pair of vertices. A graph with at most one edge per vertex pair and no self-loops is a **simple graph**; allowing repeated edges gives a **multigraph**, and a loop has both ends at the same vertex.

The **degree** $\deg(v)$ of a vertex is the number of edges incident to it (a loop counts twice). A degree-$0$ vertex is **isolated**; a degree-$1$ vertex is a **pendant**. **Handshaking theorem:**
$$\sum_{v\in V}\deg(v)=2|E|.$$
Intuitively, each edge contributes one to the degree of each of its two endpoints. **Corollary:** the number of vertices of odd degree is even (the total is even, so the odd terms must pair up).

A simple graph on $n$ vertices in which every pair is adjacent is the **complete graph** $K_n$, with $\binom{n}{2}=\dfrac{n(n-1)}{2}$ edges. If the vertices split into two parts $V_1,V_2$ with edges only between parts, the graph is **bipartite**; when the parts have sizes $m,n$ and all cross-edges are present it is $K_{m,n}$. **Characterization:** a graph is bipartite iff it contains no **odd cycle**.

Two graphs $G_1,G_2$ are **isomorphic** if there is a bijection between vertices preserving adjacency; isomorphic graphs are identical apart from how they are drawn and share every graph-theoretic property (degree sequence, connectivity, Eulerian-ness, etc.).

### Matrix Representations

Let $V=\{v_1,\dots,v_n\}$. The **adjacency matrix** $A=(a_{ij})$ is an $n\times n$ matrix: in a simple graph $a_{ij}=1$ when $v_i$ and $v_j$ are adjacent, else $a_{ij}=0$. For an undirected graph $A$ is **symmetric** ($a_{ij}=a_{ji}$), has all-zero diagonal (no loops), and the sum of row $i$ (or column $i$) equals $\deg(v_i)$.

**Meaning of powers:** the $(i,j)$ entry of $A^k$ counts the walks of **length exactly $k$** from $v_i$ to $v_j$. For instance, the diagonal entry $(i,i)$ of $A^2$ gives the number of closed walks of length $2$ through $v_i$, which equals $\deg(v_i)$ in a simple graph. This gives a uniform way to **count walks** by matrix multiplication.

Example: the triangle $K_3$ on $v_1,v_2,v_3$ has adjacency matrix
$$A=\begin{pmatrix}0&1&1\\1&0&1\\1&1&0\end{pmatrix}.$$
Each row sums to $2$, matching every vertex having degree $2$. Its square
$$A^2=\begin{pmatrix}2&1&1\\1&2&1\\1&1&2\end{pmatrix}$$
has diagonal $2=\deg(v_i)$, and each off-diagonal $1$ says any two vertices are joined by exactly one walk of length $2$ (through the third vertex).

### Walks, Circuits and Connectivity

An alternating sequence of vertices and edges $v_0,e_1,v_1,\dots,e_k,v_k$ is a **walk** of length $k$. A walk with distinct edges is a **trail**; one with distinct vertices is a **path**. A walk with equal start and end is **closed**; a closed trail is a **circuit**; a circuit with distinct vertices (except endpoints) is a **cycle**. A useful fact: if there is a walk from $u$ to $v$, there is also a **path** (delete the loop segments between repeated vertices).

If every two vertices are joined by some walk, $G$ is **connected**. A general graph decomposes into maximal connected subgraphs called **connected components**. A vertex whose removal increases the number of components is a **cut vertex**; an edge whose removal does so is a **bridge**.

### Euler and Hamiltonian Graphs

A circuit traversing **every edge exactly once** is an **Euler circuit**; a connected graph having one is an **Eulerian graph**. If only such a trail (not necessarily closed) is required, it is an **Euler trail**.

**Euler's theorem (connected undirected graph):**

- An **Euler circuit** exists $\iff$ every vertex has even degree.
- An **Euler trail** exists $\iff$ there are exactly $0$ or $2$ vertices of odd degree (when $2$, the trail must start and end at them).

Intuition: each time a trail passes through an intermediate vertex it uses two edges, so interior vertices need even degree; a closed circuit needs the start vertex to be even too. The famous **Königsberg bridges** problem has no solution precisely because all four land masses have odd degree.

A cycle visiting **every vertex exactly once** is a **Hamiltonian cycle**; a graph having one is **Hamiltonian**. Unlike the Euler case, there is **no simple characterization** for Hamiltonicity (it is NP-complete). A common **sufficient** condition is Dirac's theorem: in a simple graph with $n\ge3$, if every vertex has $\deg(v)\ge n/2$, then $G$ has a Hamiltonian cycle; the weaker Ore's theorem only needs $\deg(u)+\deg(v)\ge n$ for every pair of non-adjacent $u,v$. These are sufficient, not necessary.

### Trees and Spanning Trees

A **tree** is a connected acyclic graph; an acyclic graph not required to be connected is a **forest**. A tree on $n$ vertices has exactly $n-1$ edges, and there is a **unique** path between any two vertices. Every tree with at least $2$ vertices has at least two **leaves** (degree-$1$ vertices). The following statements about an $n$-vertex graph $T$ are **mutually equivalent** (equivalent definitions of a tree):

1. $T$ is connected and acyclic;
2. $T$ is connected with exactly $n-1$ edges;
3. $T$ is acyclic with exactly $n-1$ edges;
4. there is a unique path between any two vertices of $T$;
5. $T$ is connected, but removing any edge disconnects it (edge-minimal connected);
6. $T$ is acyclic, but adding any new edge creates a unique cycle (edge-maximal acyclic).

A **spanning tree** of a connected graph $G$ is a tree subgraph containing all vertices of $G$; every connected graph has at least one. On a weighted graph, the spanning tree of least total weight is a **minimum spanning tree (MST)**, found by **Kruskal** (add edges by increasing weight, skipping any that form a cycle, until $n-1$ are chosen) or **Prim** (start from one vertex; each step add the cheapest edge joining the chosen set to the rest). Both are greedy and provably optimal.

### Summary

- Handshaking $\sum\deg(v)=2|E|$; the number of odd-degree vertices is even.
- $K_n$ has $\binom{n}{2}$ edges; bipartite $\iff$ no odd cycle.
- Adjacency matrix is symmetric; the $(i,j)$ entry of $A^k$ = number of length-$k$ walks.
- Euler circuit $\iff$ all degrees even; Euler trail $\iff$ exactly $0$ or $2$ odd-degree vertices.
- No characterization for Hamiltonicity; Dirac $\deg\ge n/2$ and Ore $\deg u+\deg v\ge n$ are sufficient.
- Tree: connected acyclic, $n-1$ edges on $n$ vertices, unique path between any two; six equivalent forms; MST via Kruskal / Prim.

## Worked Examples

**Example 1.** A graph has $6$ vertices with degrees $4,3,3,2,2,2$. Find the number of edges and decide whether this degree sequence is possible.

> **Solution.** By handshaking, $\sum\deg(v)=4+3+3+2+2+2=16=2|E|$, so $|E|=8$. The odd-degree vertices (the two of degree $3$) number $2$, which is even, consistent with the corollary; hence the **sequence is feasible**.

**Example 2.** Is the complete graph $K_4$ Eulerian? What about $K_5$?

> **Solution.** In $K_n$ every vertex has degree $n-1$.
>
> - $K_4$: each degree $=3$, odd, so odd-degree vertices exist (in fact $4$ of them). By Euler's theorem it is **not Eulerian**, and has no Euler trail either (more than $2$ odd vertices).
> - $K_5$: each degree $=4$, all even, and $K_5$ is connected, so it **is Eulerian** with an Euler circuit.

**Example 3.** A tree has $10$ vertices — how many edges? And: if a connected graph on $10$ vertices has exactly $9$ edges, must it be a tree?

> **Solution.** A tree on $n$ vertices has exactly $n-1$ edges, so a $10$-vertex tree has $\mathbf{9}$ edges.
>
> Conversely, by the equivalent definition (statement 2), "connected + exactly $n-1$ edges" forces acyclicity, hence a tree. The given graph is connected on $10$ vertices with exactly $9=n-1$ edges, so it **must be a tree**.

**Example 4.** Use Kruskal's algorithm to find a minimum spanning tree on a weighted graph with edges $AB{=}1$, $BC{=}2$, $AC{=}2$, $CD{=}3$, $BD{=}4$.

> **Solution.** Sort edges by increasing weight: $AB(1),BC(2),AC(2),CD(3),BD(4)$. Examine each, skipping any that would form a cycle:
>
> 1. Take $AB(1)$ — forest now $\{A,B\}$.
> 2. Take $BC(2)$ — adds $C$, giving $\{A,B,C\}$.
> 3. Consider $AC(2)$ — $A,C$ are already connected, so this would close the cycle $A\!-\!B\!-\!C\!-\!A$; **discard**.
> 4. Take $CD(3)$ — adds $D$, giving $\{A,B,C,D\}$.
>
> We have chosen $3=n-1$ edges ($n=4$), so stop. The MST is $\{AB,BC,CD\}$ with total weight $1+2+3=6$.

**Example 5.** Prove that any simple graph with at least two vertices has two vertices of the same degree.

> **Solution.** Let the graph have $n\ge2$ vertices. Each vertex degree is one of $0,1,\dots,n-1$, that is $n$ possible values. But $0$ and $n-1$ **cannot both occur**: if some vertex has degree $n-1$ (adjacent to all others), then no vertex is isolated (degree $0$); conversely, if some vertex has degree $0$, no vertex can reach degree $n-1$.
>
> Hence at most $n-1$ degree values are actually available, while there are $n$ vertices. By the **pigeonhole principle**, putting $n$ vertices into at most $n-1$ "degree boxes" forces two vertices to share a degree.

## Exercises

1. A simple graph has $7$ vertices, each of degree $4$. Find the number of edges.
2. Write the adjacency matrix of the path graph $P_3$ (vertices $v_1\!-\!v_2\!-\!v_3$, two edges) and verify each row sum equals the degree of the corresponding vertex.
3. A connected graph has $4$ vertices of odd degree. Does it have an Euler trail? An Euler circuit? Justify.
4. How many edges does $K_5$ have? How many edges does a spanning tree of it have?
5. A simple graph has degree sequence $5,3,2,2,1$. Decide whether it is possible (hint: use the handshaking corollary first).
6. How many edges does the complete bipartite graph $K_{3,3}$ have? Is it Eulerian? Does it have a Hamiltonian cycle?

**Answers.** 1) $\sum\deg=7\times4=28=2|E|$, so $|E|=14$; 2) $A=\begin{pmatrix}0&1&0\\1&0&1\\0&1&0\end{pmatrix}$, row sums $1,2,1$, exactly $\deg(v_1)=1,\deg(v_2)=2,\deg(v_3)=1$; 3) with $4$ odd-degree vertices ($>2$), by Euler's theorem there is **neither an Euler trail nor an Euler circuit**; 4) $K_5$ has $\binom{5}{2}=10$ edges, and its spanning tree (on $5$ vertices) has $5-1=4$ edges; 5) $\sum\deg=5+3+2+2+1=13$ is **odd**, contradicting handshaking ($\sum\deg=2|E|$ must be even), so the sequence is **impossible**; 6) $K_{3,3}$ has $3\times3=9$ edges; each vertex has degree $3$ (odd) with $6$ odd-degree vertices, so it is **not Eulerian**; but it **does have a Hamiltonian cycle**, e.g. $a_1\!-\!b_1\!-\!a_2\!-\!b_2\!-\!a_3\!-\!b_3\!-\!a_1$ (parts labeled $a_i,b_i$).
